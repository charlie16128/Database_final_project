# 寵物用品循環平台 Database Final Project 計畫書

## 一、專案架構總覽

本專案是一個以 **Vue 3 + Vite** 製作前端畫面，搭配 **Node.js + Express** 作為後端 API，並使用 **SQLite** 作為資料庫的寵物用品循環平台。
系統主要提供會員註冊登入、二手寵物用品刊登、商品留言、私訊聊天室、永續指南管理與商品搜尋功能。

## 二、技術選型

| 類別 | 技術 | 用途 |
|---|---|---|
| 前端框架 | Vue 3 + Vite | 製作 SPA 網頁 |
| 前端路由 | Vue Router | 控制頁面切換 |
| 後端 | Node.js + Express | 提供 API 給 Vue 呼叫 |
| 資料庫 | SQLite | 儲存會員、商品、留言、聊天室資料 |
| 密碼加密 | bcrypt | 將密碼 hash 後存入資料庫 |
| 登入驗證 | JWT | 維持登入狀態 |
| 圖片上傳 | multer | 將圖片存到 server/uploads |
| API 呼叫 | fetch | Vue 呼叫 Express API |

建議安裝套件：

```bash
npm install vue-router express better-sqlite3 cors bcrypt jsonwebtoken multer
npm install -D nodemon concurrently
```

## 三、系統目標

本平台的主要目標是建立一個寵物用品循環再利用平台，讓飼主可以刊登閒置用品、與其他會員交流、透過私訊聯絡商品擁有者，並閱讀永續飼養相關知識。

主要功能：

1. 會員註冊、登入、登出
2. 密碼使用 bcrypt hash 後存入 SQLite
3. 二手寵物用品新增、修改、刪除、查詢
4. 商品狀態管理：刊登中、已預訂、已完成
5. 商品圖片上傳至 `server/uploads`
6. 商品頁留言功能
7. 私訊聊天室功能
8. 永續指南新增、修改與顯示
9. RWD 響應式外觀設計

## 四、前後端運作流程

Vue 不直接操作 SQLite，而是透過 API 與後端溝通。

```text
使用者操作 Vue 頁面
        ↓
Vue 使用 fetch 呼叫 API
        ↓
Express 接收請求
        ↓
驗證 JWT / 檢查資料
        ↓
操作 SQLite
        ↓
回傳 JSON 給 Vue
        ↓
Vue 更新畫面
```

範例：

```js
const response = await fetch('/api/items')
const items = await response.json()
```

登入後呼叫需要權限的 API：

```js
await fetch('/api/items', {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${localStorage.getItem('token')}`
  },
  body: formData
})
```

## 五、資料庫設計

### 1. users 會員資料表

用來儲存會員帳號資料。

```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT NOT NULL UNIQUE,
  email TEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

| 欄位 | 說明 |
|---|---|
| id | 會員編號 |
| username | 使用者名稱 |
| email | 電子信箱 |
| password_hash | bcrypt 加密後密碼 |
| created_at | 註冊時間 |

---

### 2. items 商品資料表

用來儲存二手寵物用品資料。

```sql
CREATE TABLE items (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  title TEXT NOT NULL,
  category TEXT NOT NULL,
  description TEXT,
  condition TEXT,
  status TEXT DEFAULT '刊登中',
  image_path TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

商品狀態：

```text
刊登中
已預訂
已完成
```

| 欄位 | 說明 |
|---|---|
| user_id | 商品擁有者 |
| title | 商品名稱 |
| category | 商品分類 |
| description | 商品描述 |
| condition | 物品狀況 |
| status | 商品狀態 |
| image_path | 圖片路徑，例如 `/uploads/cat-toy.jpg` |

---

### 3. item_comments 商品留言資料表

用來儲存商品頁留言。

```sql
CREATE TABLE item_comments (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  item_id INTEGER NOT NULL,
  user_id INTEGER NOT NULL,
  content TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (item_id) REFERENCES items(id),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

---

### 4. guides 永續指南資料表

用來儲存寵物用品維護、再利用與回收知識。

```sql
CREATE TABLE guides (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  title TEXT NOT NULL,
  category TEXT NOT NULL,
  content TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

---

### 5. chat_rooms 聊天室資料表

當使用者在商品頁點擊「聯絡擁有者」時，系統會建立或取得聊天室。

```sql
CREATE TABLE chat_rooms (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  item_id INTEGER NOT NULL,
  buyer_id INTEGER NOT NULL,
  owner_id INTEGER NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (item_id) REFERENCES items(id),
  FOREIGN KEY (buyer_id) REFERENCES users(id),
  FOREIGN KEY (owner_id) REFERENCES users(id)
);
```

設計邏輯：

```text
同一個商品 + 同一個買家 + 同一個擁有者
只建立一個聊天室
```

---

### 6. chat_messages 聊天訊息資料表

用來儲存聊天室內的訊息。

```sql
CREATE TABLE chat_messages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  room_id INTEGER NOT NULL,
  sender_id INTEGER NOT NULL,
  message TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (room_id) REFERENCES chat_rooms(id),
  FOREIGN KEY (sender_id) REFERENCES users(id)
);
```

## 六、API 設計

### 1. Auth API

| 方法 | 路徑 | 功能 |
|---|---|---|
| POST | `/api/auth/register` | 註冊會員 |
| POST | `/api/auth/login` | 登入 |
| GET | `/api/auth/me` | 取得目前登入者 |

註冊流程：

```text
檢查 email 是否重複
→ 使用 bcrypt hash 密碼
→ 寫入 users
→ 回傳成功訊息
```

登入流程：

```text
查詢 email
→ bcrypt.compare 檢查密碼
→ 成功後產生 JWT
→ 回傳 token 與會員資料
```

---

### 2. Item API

| 方法 | 路徑 | 功能 |
|---|---|---|
| GET | `/api/items` | 取得商品列表 |
| GET | `/api/items/:id` | 取得商品詳細資料 |
| POST | `/api/items` | 新增商品 |
| PUT | `/api/items/:id` | 修改商品 |
| DELETE | `/api/items/:id` | 刪除商品 |
| GET | `/api/my/items` | 查看我的商品 |

新增商品流程：

```text
檢查 JWT
→ 接收表單資料與圖片
→ 圖片存到 server/uploads
→ SQLite 儲存商品資料與圖片路徑
→ 回傳新增成功
```

修改商品流程：

```text
檢查 JWT
→ 確認商品 user_id 是否為目前登入者
→ 更新商品資料
→ 回傳修改成功
```

---

### 3. Comment API

| 方法 | 路徑 | 功能 |
|---|---|---|
| GET | `/api/items/:id/comments` | 取得商品留言 |
| POST | `/api/items/:id/comments` | 新增商品留言 |
| DELETE | `/api/comments/:id` | 刪除自己的留言 |

---

### 4. Guide API

| 方法 | 路徑 | 功能 |
|---|---|---|
| GET | `/api/guides` | 取得永續指南列表 |
| GET | `/api/guides/:id` | 取得指南詳細資料 |
| POST | `/api/guides` | 新增指南 |
| PUT | `/api/guides/:id` | 修改指南 |
| DELETE | `/api/guides/:id` | 刪除指南 |

---

### 5. Chat API

| 方法 | 路徑 | 功能 |
|---|---|---|
| POST | `/api/chats/start` | 建立或取得聊天室 |
| GET | `/api/chats` | 取得我的聊天室列表 |
| GET | `/api/chats/:roomId/messages` | 取得聊天室訊息 |
| POST | `/api/chats/:roomId/messages` | 傳送訊息 |

聯絡擁有者流程：

```text
使用者在商品頁點擊「聯絡擁有者」
→ 前端呼叫 POST /api/chats/start
→ 後端檢查是否已有聊天室
→ 若沒有則建立 chat_rooms
→ 回傳 room_id
→ Vue Router 跳轉到 /messages/:roomId
```

## 十一、核心邏輯實作說明

### 1. 登入驗證邏輯

後端：

```text
使用者登入
→ 查詢 users.email
→ 使用 bcrypt.compare 驗證密碼
→ 成功後產生 JWT
→ 回傳 token
```

前端：

```text
收到 token
→ 存到 localStorage
→ 往後 API request 都帶 Authorization header
```

---

### 2. 商品權限邏輯

只有商品擁有者可以修改或刪除商品。

```text
PUT /api/items/:id
→ 檢查 JWT
→ 取得目前 user_id
→ 查詢商品 user_id
→ 若不是本人，拒絕操作
→ 若是本人，允許修改
```

---

### 3. 圖片上傳邏輯

```text
前端使用 FormData
→ 後端 multer 接收圖片
→ 圖片存到 server/uploads
→ SQLite 儲存圖片路徑
```

前端送出格式：

```js
const formData = new FormData()
formData.append('title', title)
formData.append('category', category)
formData.append('description', description)
formData.append('image', imageFile)
```

---

### 4. 聊天室建立邏輯

```text
POST /api/chats/start
body: { item_id }

後端：
1. 根據 item_id 查詢商品擁有者 owner_id
2. buyer_id 來自 JWT
3. 若 buyer_id 等於 owner_id，不允許聯絡自己
4. 查詢是否已有相同 item_id + buyer_id + owner_id 的聊天室
5. 有則回傳原 room_id
6. 沒有則建立新聊天室並回傳 room_id
```

---

### 5. 商品留言邏輯

```text
會員登入後才能留言
留言會綁定 item_id 與 user_id
商品詳細頁載入時同步載入留言
```
